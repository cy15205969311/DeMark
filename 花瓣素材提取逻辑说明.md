# 花瓣素材提取逻辑说明

本文档用于说明当前仓库里花瓣网素材提取能力的实现方式、接入入口、结果结构、限制边界和调试建议。

这份文档描述的是当前已经落地的代码行为，重点对应以下文件：

- `core/image_extractor.py`
- `crawlers/huaban_crawler.py`
- `utils/url_parser.py`
- `utils/image_validator.py`
- `utils/downloader.py`
- `gui/main_window.py`
- `tests/test_huaban.py`

如果后续花瓣公开接口字段变化，或我们继续增加“授权账号下载”等能力，优先同步更新本文档。

## 1. 功能目标

当前花瓣提取逻辑的目标不是“绕过权限拿无水印原素材”，而是把公开可访问结果稳定整理出来，并清楚地区分：

1. 公开可下载的普通图片
2. 只能拿到带水印公开预览图的商用素材
3. 连公开预览图也无法稳定返回的受限素材

当前实现目标包括：

1. 正确识别 `huaban.com` / `api.huaban.com` / `huabanimg.com`
2. 自动清理分享文案后缀，只保留实际链接
3. 优先走花瓣公开 JSON 接口，不经过第三方网关
4. 支持 `pin / board / discovery` 三类链接
5. 当公开接口只能返回带水印预览图时，继续允许下载预览图，但明确标注风险
6. 仅在花瓣专用提取器和花瓣相关公共逻辑内修改，不影响其他平台

## 2. 当前代码入口

### 2.1 平台接入入口

花瓣平台是在 `core/image_extractor.py` 中接入统一调度的。

关键点：

- `ImageExtractor.__init__()` 中注册 `HuabanCrawler`
- `_extract_local()` 中识别 `platform == 'Huaban'`
- `_extract_huaban()` 中把请求转交给 `HuabanCrawler.extract_image()`
- 对花瓣显式跳过第三方 API 网关，直接进入官方公开接口路径

### 2.2 URL 解析入口

花瓣链接解析在 `utils/url_parser.py` 中完成，主要负责两件事：

1. 平台识别：把 `huaban.com` / `api.huaban.com` / `huabanimg.com` 识别为 `Huaban`
2. 分享链接清洗：去掉分享文案、fragment 和无关尾巴

例如这类输入：

```text
https://huaban.com/pins/6860695449?from=share&share_text=
这张来自「花瓣网」的图不错哦，快来看看！
```

会被清洗成：

```text
https://huaban.com/pins/6860695449?from=share&share_text=
```

## 3. 整体提取流程

当前花瓣提取流程可以概括为：

1. URL 清洗与平台识别
2. 在 `ImageExtractor.extract_image()` 中识别为 `Huaban`
3. 跳过第三方 API 网关
4. 进入 `HuabanCrawler.extract_image()`
5. 根据链接类型进入 `pin / board / discovery`
6. 把结果统一整理成单图或多图结构
7. GUI 根据结果状态显示成功卡片、警告卡片或受限提示卡片

## 4. 花瓣提取器的实现策略

主逻辑在 `crawlers/huaban_crawler.py` 的 `HuabanCrawler.extract_image()`。

当前实现不直接抓取花瓣保护较强的 HTML，而是优先使用花瓣公开 JSON 接口：

- `https://api.huaban.com/pins/{pin_id}`
- `https://api.huaban.com/boards/{board_id}`
- `https://api.huaban.com/boards/{board_id}/pins`
- `https://api.huaban.com/discovery...`

这样做的原因是：

- 结构更稳定
- 结果更容易标准化
- 不依赖浏览器执行
- 更适合 pin / board / discovery 三类不同页面

## 5. 支持的链接类型

函数：`HuabanCrawler._classify_link()`

当前会把花瓣链接分为以下几类：

- `pin`
- `board`
- `discovery`
- `unknown`

### 5.1 pin

例如：

```text
https://huaban.com/pins/7083303792
```

提取逻辑：

1. 从路径中取出 `pin_id`
2. 请求 `api.huaban.com/pins/{pin_id}`
3. 从返回的 `pin.file.key` 或 `pin.file.url` 组装最终图片地址

### 5.2 board

例如：

```text
https://huaban.com/boards/77052553
```

提取逻辑：

1. 请求 `boards/{board_id}` 拿画板元信息
2. 请求 `boards/{board_id}/pins` 拿当前画板中的 pin 列表
3. 从多个 pin 中筛出有效图片
4. 组装成多图结果

### 5.3 discovery

例如：

```text
https://huaban.com/discovery
https://huaban.com/discovery?limit=12
```

提取逻辑：

1. 转换成 `api.huaban.com/discovery...`
2. 读取返回的 pin 列表
3. 过滤、校验后组装成多图结果

## 6. 图片地址如何生成

函数：`HuabanCrawler._normalize_pin_image_url()`

当前优先使用：

- `pin.file.key`

并把它转换成：

```text
https://hbimg.huabanimg.com/{file.key}
```

如果没有 `file.key`，则回退使用 `pin.file.url` 中的 path，最终同样统一到：

```text
https://hbimg.huabanimg.com/{path}
```

这样做的好处是：

- 避免使用带签名参数的边缘域名
- 下载器和校验器可以统一处理 `hbimg.huabanimg.com`
- 更适合后续去重与测试

## 7. 公开商用素材的两种受限状态

花瓣商用素材经常不是“完全提取失败”，而是“公开接口只给预览图”。

当前实现把这类情况分成两个状态。

### 7.1 `preview_image`

含义：

- 公开接口没有给无水印原素材下载字段
- 但仍然给出了一个可访问的公开预览图地址
- 这个预览图通常可能带平台水印

当前行为：

- 返回 `imageUrl / imageUrls / pages / pageCount`
- 同时附带：
  - `status: "preview_image"`
  - `warningText`
  - 详细 `error` 原因
- GUI 显示为黄色警告卡片
- 允许继续下载这个预览图

用户侧文案是：

```text
已提取公开预览图，可直接下载，但可能包含平台水印，未获取到无水印原素材
```

### 7.2 `preview_only`

含义：

- 公开接口没有给无水印原素材下载字段
- 连公开预览图也无法稳定校验通过

当前行为：

- 返回 `status: "preview_only"`
- 不返回 `imageUrl`
- GUI 显示黄色警告卡片
- 只保留“复制原链 / 打开原页”这类操作

## 8. 为什么会被判定为公开预览素材

函数：`HuabanCrawler._is_public_preview_only_pin()`

当前主要判断以下信号：

1. `extra.material_id` 或 `file_material.material_id` 存在
2. `file_material.source_type == "produced"`
3. `source / link / original` 都为空

也就是说，这类 pin 更像“花瓣站内商用素材”，而不是用户外链图片或开放原图资源。

当命中这些条件时，系统不会把它误报成“无水印成功提取”。

## 9. 图片有效性校验与下载

### 9.1 校验

图片有效性校验依赖 `utils/image_validator.py`。

对花瓣相关域名会自动补：

- `Referer: https://huaban.com/`
- `Origin: https://huaban.com`

目的是降低：

- 防盗链失败
- HEAD / GET 被拒
- 误判为“图片地址失效”

### 9.2 下载

下载逻辑在 `utils/downloader.py`。

对花瓣图片域名会额外补：

- `Referer`
- `Origin`
- `Sec-Fetch-*`

这样即使下载的是公开预览图，也能更稳定地落盘。

如果原始 URL 没有扩展名，会根据 `Content-Type` 推断 `.png / .jpg / .webp`。

## 10. GUI 层对花瓣的处理

GUI 逻辑在 `gui/main_window.py`。

当前花瓣相关处理重点如下：

1. 平台选择里新增 `🌸 花瓣`
2. 自动检测支持 `huaban.com` / `huabanimg.com`
3. `preview_image` 显示为黄色警告卡片，但保留：
   - `复制预览图`
   - `查看预览图`
   - `下载预览图`
   - `打开原页`
4. `preview_only` 显示为黄色警告卡片，只保留：
   - `复制原链`
   - `打开原页`
5. 状态栏单独统计为 `警告`，不计入真正的 `失败`

## 12. 当前测试覆盖

花瓣相关测试位于 `tests/test_huaban.py`。

当前重点覆盖：

1. pin / board / discovery 链接识别
2. `hbimg.huabanimg.com` 原始地址生成
3. 公开商用素材识别
4. discovery / board 多图组装
5. 公开预览图结果组装
6. 主提取器对花瓣跳过第三方网关
7. URL 清洗
8. 花瓣防盗链请求头
9. 下载阶段扩展名修正

建议本地优先运行：

```bash
python -m pytest tests/test_huaban.py -q
```

## 13. 边界与限制

这一部分很重要。

当前花瓣实现的边界是：

1. 支持稳定提取公开接口暴露的图片结果
2. 支持把“公开预览图”和“无水印原素材”明确区分开
3. 不会把带水印预览图误报成无水印成功结果

当前不做的事情包括：

1. 不尝试绕过花瓣授权限制获取无水印原素材
2. 不尝试去除花瓣商用素材水印
3. 不把 DOM 级遮罩移除视为正式提取路径

如果后续需要无水印原素材，合规方向通常是：

1. 用户使用有权限的花瓣账号按官方方式下载
2. 后续增加“授权账号登录态 / Cookie 下载”支持

## 14. 维护建议

维护花瓣提取器时，建议遵循以下原则：

1. 优先在 `crawlers/huaban_crawler.py` 内解决问题
2. 只有公共防盗链或下载问题时，才动 `utils/image_validator.py` 和 `utils/downloader.py`
3. 每修一次真实链接问题，至少补一条 `tests/test_huaban.py`
4. 不要把“能拿公开预览图”误写成“能拿无水印原图”
5. UI 文案必须明确说明素材是否只是公开预览图

## 15. 结论

当前花瓣提取逻辑的核心原则可以概括为一句话：

优先使用花瓣公开接口稳定拿到公开可访问结果，并把“普通成功”“公开预览图可下载”“仅有受限提示”三类状态明确区分开。

这样做的好处是：

- 改动范围局限在花瓣提取器和花瓣相关公共逻辑
- 不会影响其他平台
- 日志、测试和 GUI 表现一致
- 不会把带水印预览图误报成无水印原素材
